# Euro-Legend V1.2 — iPhone Layout Fix + Ambient Music

V1.1 shipped the polish. V1.2 fixes the issues surfaced on real-device
testing (iPhone 16 Pro, iOS 26) and adds an in-browser dark-minimal-techno
soundtrack so the app actually feels like exploring at night.

**Build:** `npm run build` ✅ green — 319.96 kB JS (92.15 kB gzip) ·
15.82 kB CSS (4.13 kB gzip). 1500 modules. No warnings. Built in 2.5 s.
**No new runtime dependencies** — music is generated via the WebAudio
API that already shipped in V1.

### Fix 3 — Map edges

`NAV_OVERSCAN` went from 2.5x to 3.0x. The SVG element therefore renders
at 300% of its container with a -100% offset, which gives roughly 70%
extra margin on every side after any 0–360° rotation + 28° tilt. The
underlying container retains `background: palette.mapBg` so even in the
pathological case where the SVG ever fails to cover, the user sees the
day cream / night indigo of the map, not the page background.

### Fix 7 — Ambient music

The new `MusicEngine` (`src/App.jsx`, just before `useOrientation`) is a
single self-contained module that schedules an 8-bar, 125-BPM, A-minor
loop using only the WebAudio primitives already in the project. **Zero
new dependencies, zero audio files, zero copyright exposure.**

Layers per bar:

1. **Kick** — sine 110→38 Hz with 0.42 s exp decay, on every quarter.
2. **Closed hihat** — filtered noise burst, dotted-8th grid, panned ±0.22 alternating.
3. **Bass** — sawtooth A2 / C3 / G2, syncopated 16ths, lowpass 360→180 Hz.
4. **Pluck motif** — triangle pulses on A4 · C5 · E5 · A4 · G4 · E5 · A4 · rest, A-minor pentatonic, every 4 bars, panned, lowpass 2200→700 Hz.
5. **Bell** — sine 1318.51 Hz on beat 3 of bar 8, fed through the bus convolver for tail.

Master chain: compressor (-18 dB / 12 dB knee / 3.5:1) → master gain →
parallel dry (78%) + convolver (22%, algorithmic 1.6 s noise impulse) →
destination. The look-ahead scheduler refills the next 8-bar slot every
250 ms, so a tab in the background that survives the gap still resyncs
on `visibilitychange` resume.

UX wiring:

- New `<Music />` button in the portrait header AND in the landscape
  sidebar's bottom cluster. Off → palette icon bg; on → cyan gradient
  with a 14 px cyan glow.
- **iOS gesture rule respected:** the toggle always calls
  `getAudioCtx()` first (a user gesture), then flips
  `settings.musicOn`. No autoplay attempted anywhere.
- **Persisted:** `soundOn`, `musicOn`, `musicVolume` all live in
  `euro-legend-v1` localStorage. Defaults: `musicOn: false`,
  `musicVolume: 0.35`.
- **Volume slider** appears under the music toggle in Settings when
  music is on (preferred location — sliders inline in the header
  are too easy to scrub by accident on a touch device).
- **Ducks** to 40% when any modal is open ("duck under voiceover").
- **Pauses** when navigating back to City Atlas and on
  `visibilitychange`. Resumes when both gates clear AND `musicOn`.
- **Spotify deep-link** in Settings → "Soundtrack reference":
  links to a Spotify search for "Boris Brejcha Never Look Back" so the
  user can play the real track in another app while our loop carries
  the under-walk vibe.

## Decisions made (no user available)

- **Music engine = WebAudio, not Tone.js.** Tone.js would have added
  ~50 kB gzipped for what is, in the end, five oscillator voices and a
  noise burst. The existing `getAudioCtx()` is reused, so the engine
  shares the same context that already powers SFX.
- **No `public/sounds/*.ogg` fallback was generated.** OPTION B in the
  brief was the safety net if Tone.js generation failed; with the
  WebAudio path working cleanly there's nothing to fall back to.
- **Volume slider in Settings, not behind a long-press.** Long-press
  detection is fiddly on iOS and tends to trigger the system context
  menu over a button. The Senses section in Settings is the natural
  home and it already had Sound and Haptics — Music slots in next to
  them.
- **Spotify link = search URL, not a specific track URI.** Linking
  `spotify:track:<id>` requires the canonical track ID and the search
  URL gracefully falls back to a result list for users who don't have
  Spotify installed. Brand-safe + works for everyone.
- **Sidebar width `clamp(220px, 30vw, 280px)`.** 280 px is the spec
  ceiling; 220 px is the floor to keep the icon cluster legible on
  iPhone SE landscape (667 × 375).
- **Header is hidden in landscape**, not relocated. Spec said the back
  arrow should float over the map's top-left and header buttons should
  cluster at the bottom of the sidebar. Re-rendering the same
  `<header>` in both layouts would have meant a separate container per
  orientation; hiding it and rendering a small `LandscapeChrome`
  component is fewer lines and avoids duplicate React state.
- **Stars strip in portrait is one button, not five.** Tapping any
  star previously opened that landmark's modal. The new compact strip
  tap-targets the whole row and opens the Achievements modal, which
  has the proper landmark tiles. Saves real-estate AND gives a
  cleaner 44 px hit target.
- **NAV_OVERSCAN bumped to 3.0** instead of being kept at 2.5 with an
  added container background. Both were tried; raising the SVG
  overscan also helps the cobblestones/buildings sit consistently
  through the full rotation curve.
- **Mini-map building polygons removed.** At 110 px they read as
  illegible smudges. The simplified street network alone gives enough
  shape recognition.

## Known limitations / followups

- **Music will not start on a fresh PWA launch until the user taps the
  Music toggle.** This is a hard iOS Safari requirement (`AudioContext`
  cannot be resumed without a user gesture). Documented; no workaround
  exists.
- **The music loop is short on its own** — 15.36 s seamless. After ~5
  loops the pluck motif's predictability becomes noticeable. A V1.3
  pass could rotate two or three pluck motifs across phrases for
  variety, but the brief was an under-the-walk ambient bed, not a DJ
  set — so the repetition is intentional and matches the genre.
- **No master limiter past the compressor.** If the user pushes
  `musicVolume` to 1.0 AND a SFX fires (chime, level-up) at the same
  time, the sum can clip on weak DACs. Kept default volume at 0.35 and
  the music master gain at the compressor — adequate for normal use.
- **Spotify link opens in a new tab** rather than the native Spotify
  app even on iOS. A `spotify:track:` URI would do that but requires
  the canonical track ID; the search URL is universal and degrades
  gracefully when Spotify isn't installed.
- **Landscape with the standard header hidden** means users who reach
  the explore screen via portrait, rotate to landscape, then look for
  the chapter label ("Chapter I · Altstadt") will not see it. The back
  arrow is the only chrome up top. Acceptable trade-off; the label is
  visible in the Settings modal under About.
- **The compact tier pill drops the "Next tier · 🏛 Eurocrat · 38%"
  hint** that was on the old card. That hint still appears in the full
  Tier modal (tap the pill). Acceptable trade-off for not clipping.
